> # 🛒 Commerce Project 🛒

> ## 👨‍🏫 프로젝트 소개

#### 이 프로젝트는 객체지향 설계를 기반으로 상품, 카테고리 등의 객체를 정의한다.
#### 각 객체 간 역할과 책임을 나누어 협력하는 구조로 구현한 커머스 플랫폼이다.

> ## 📌 주요기능 

### STEP1. 객체 지향 설계 적용 사품관리 시스템 프로그래밍
 - Product 클래스 생성 및 상품명, 가격, 설명, 재고수량 필드 정의
 - 상품 정보 출력 및 0 입력 시 프로그램 종료

### STEP2. 
 - CommerceSystem 클래스 생성
   - start 함수 구현하여 입력과 반복문 로직 구현 

### STEP3.4. 상품 선택 및 출력 기능 구현
 - 카테고리 선택 후 상품 목록 출력
 - 상품 선택 시 상세 정보 출력
 - 사용자 입력 검증 로직 추가

### Lv1.장바구니 및 주문하기 기능 추가
 - 카테고리별 상품 목록 조회
 - 상품 선택 및 상제 정보 확인
 - 장바구니 담기 기능
 - 동일 상품 재선택 시 수량 증가 처리
 - 장바구니 내역 및 총 주문 금액 확인
 - 주문 확정 시 재고 차감
 - 주문 취소 가능

> ## 🔥 Trouble Shooting
 ### 1. 출력 형식 정리
- `%d` : 정수 출력
- `%s` : 문자열 출력
- `%-15s` : 문자열 왼쪽 정렬 후 15칸 확보
- `%,d` : 천 단위 콤마 적용
- `%n` : 줄바꿈

👉 자세한 내용은 블로그 참고  
https://velog.io/@gpekd5/Java-%EB%AC%B8%EB%B2%95-printf

 ### 2. 입력값 예외 처리 문제
 - 문제 : 숫자를 입력해야 하는 상황에서 문자 입력 시 프로그램 종료
 - 원인 : `Scanner.nextInt()` 사용 시 `InputMismatchException` 발생
 - 해결 : `try-catch`를 활용하여 예외 처리 및 입력 버퍼 초기화

👉 자세한 내용은 블로그 참고  
https://velog.io/@gpekd5/Java-%EB%AC%B8%EB%B2%95-%EC%98%88%EC%99%B8%EC%B2%98%EB%A6%AC-try-catch

 ### 3. final 적용 기준에 대한 문제
 - 문제 : 처음에는 `final`을 어디에 적용해야 하는지 명확한 기준 없이 사용

 - 해결 : `final`은 “변하지 않는 값”에 사용하는 것이라는 개념을 다시 정리하고  
  상품 정보(이름, 가격, 설명)와 같이 변경되지 않는 필드에 적용

👉 자세한 내용은 블로그 참고  
https://velog.io/@gpekd5/Java-%EB%AC%B8%EB%B2%95-final-%ED%82%A4%EC%9B%8C%EB%93%9C

 ### 4. 전체 구조 설계에 대한 고민

- 문제  
  초기에는 상품, 장바구니, 수량 데이터를 하나의 흐름으로 처리하려다 보니  
  클래스 간 역할이 명확하게 나뉘지 않아 구조 설계에 어려움을 겪었다.


- 원인  
  객체지향 설계 경험이 부족하여  
  각 클래스가 어떤 책임을 가져야 하는지에 대한 이해가 부족했음


- 해결  
  예시와 코드 흐름을 따라가며 각 클래스 역할을 나누어보았다.
   - `Product` : 상품 정보
   - `CartItem` : 상품 + 수량
   - `Cart` : 장바구니 목록 및 주문 관리
   - `Category` : 카테고리 및 상품 목록


- 느낀점   
  클래스간 관계를 정의하였으나, 아직까지는 더 많은 경험을 통해 객체지향적인 사고를 가져야겠다.
